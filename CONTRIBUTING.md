# Contributing to OpenLinuxManager

First off, thank you for considering contributing to OpenLinuxManager! It's people like you that make OpenLinuxManager such a great tool.

## How Can I Contribute?

### Reporting Bugs
*   Use a clear and descriptive title for the issue.
*   Describe the exact steps which reproduce the problem in as many details as possible.
*   Include screenshots or recordings if relevant.

### Suggesting Enhancements
*   Use a clear and descriptive title.
*   Provide a step-by-step description of the suggested enhancement.
*   Explain why this enhancement would be useful to most OpenLinuxManager users.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Issue that pull request!

## Styleguide

### Git Commit Messages
*   Use the present tense ("Add feature" not "Added feature").
*   Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
*   Limit the first line to 72 characters or less.

### TypeScript / NestJS / React
*   Follow the standard ESLint configuration provided in the project.
*   Use functional components for React.
*   Use descriptive names for variables and functions.

## Community
*   Be kind and respectful to others.
*   Follow our [Code of Conduct](CODE_OF_CONDUCT.md).
